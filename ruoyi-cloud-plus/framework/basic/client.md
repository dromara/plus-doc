## 客户端字段说明

![输入图片说明](https://foruda.gitee.com/images/1690961819029076660/c44374ac_4959041.png "屏幕截图")

| 字段名称       | 取值说明               | 注意事项               |
|------------|--------------------|--------------------|
| 客户端id      | 后端生成，用于登录校验与接口数据加密 | 无法修改，不要删除默认数据      |
| 客户端key     | 前端自定义              | 无法修改，不要删除默认数据      |
| 客户端秘钥      | 前端自定义              | 无法修改，不要删除默认数据      |
| 授权类型       | 密码/短信/邮件/小程序/第三方认证 | 决定该客户端支持的登录方式      |
| 设备类型       | PC 端 / APP 端       | -                  |
| Token 活跃超时 | 秒                  | 无操作即过期，默认 1800 秒   |
| Token 固定超时 | 秒                  | 固定过期时间，默认 604800 秒 |

## 使用新的客户端 ID

步骤：

1. 在客户端管理页面生成新的客户端 ID
2. 将 ID 写入前端配置文件

通常对应前端环境变量 `VITE_APP_CLIENT_ID`，如果使用默认 `plus-ui`，优先从环境配置文件统一修改，不要散落在业务代码里硬编码。

### 2. 配置前端请求头

需要在全局请求头中增加 `clientid`。  
可参考前端 `request.ts`：

![输入图片说明](https://foruda.gitee.com/images/1690965768235114596/980b88d2_4959041.png "屏幕截图")

`VITE_APP_CLIENT_ID` 为配置文件中的客户端 ID。

**注意：不同客户端登录后获取的 token 互不通用。**

补充说明：
- Cloud 登录入口在认证中心，但客户端 ID 的使用方式与 Vue 版本一致
- `plus-ui` 的请求头、SSE、WebSocket 等都会带上这个 `clientid`
- 如果更换了客户端 ID，前端环境变量和认证中心客户端配置要一起调整
