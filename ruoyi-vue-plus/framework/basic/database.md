# 关于数据库设计
- - -


## 表设计必备字段

参考框架基础实体类`BaseEntity`内字段

补充说明：
- 如果业务实体继承框架的 `BaseEntity`，下面这些审计字段会按约定参与自动填充
- 如果你不继承 `BaseEntity`，也建议沿用相同字段命名，便于和框架能力、代码生成保持一致

| 字段          | 作用                |
|-------------|-------------------|
| create_dept | 创建部门，用于数据归属与审计    |
| create_by   | 创建人，用于数据审计        |
| create_time | 创建时间，用于审计、排序与排查问题 |
| update_by   | 修改人，用于数据审计与问题排查   |
| update_time | 修改时间，用于数据审计与问题排查  |

## 可选字段

| 字段        | 作用   | 说明                          |
|-----------|------|-----------------------------|
| tenant_id | 租户ID | 标记数据所属租户；不需要多租户可在配置中关闭或放行单表 |
| del_flag  | 删除标记 | 逻辑删除字段，建议数据库默认值为 `0`        |
| version   | 乐观锁  | 用于标记数据版本，建议默认值从 `0` 开始      |

补充说明：
- 多租户业务表建议直接按 `TenantEntity` 约定保留 `tenant_id`
- `del_flag`、`version` 是否启用取决于你的业务需求，但字段名尽量保持一致
- 纯关系表、纯中间表如果没有审计诉求，可以按实际情况裁剪字段

## 建议规范

- 如果计划使用代码生成器，建议优先使用框架默认字段名，如 `create_by`、`create_time`、`update_by`、`update_time`、`tenant_id`、`del_flag`、`version`
- 这些字段名已经被生成器识别，生成列表、表单、查询条件时会自动做基础排除和处理
