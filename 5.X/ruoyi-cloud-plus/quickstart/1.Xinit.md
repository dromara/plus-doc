# 1.X项目初始化
- - -

## 分支选择

| 分支       | 用途    | 说明             |
|----------|-------|----------------|
| master   | 稳定发布  | 1.X 主分支        |
| dev      | 开发分支  | 代码随时更新，不建议直接使用 |
| 2.X      | 大版本分支 | 2.X 新版本        |
| future/* | 预览分支  | 新功能预览，稳定性不保证   |

## 运行环境清单
> 推荐使用 `Docker` 安装，项目内置编排文件。

| 组件         | 版本建议          | 说明                      |
|------------|---------------|-------------------------|
| JDK        | 8 / 11        | 不支持 17；JDK8 不高于 `8u202` |
| MySQL      | 5.7 / 8.0     | 5.6 未适配可能有问题            |
| Oracle     | 11g / 12c     | 其他版本未测试                 |
| PostgreSQL | 13 / 14       | 其他版本未测试                 |
| Redis      | >= 5.x        | 框架大量使用新特性               |
| MinIO      | 按需            | 支持 S3 协议的云存储也可替代        |
| Maven      | 3.6.3 / 3.8.x | 构建工具                    |
| Node.js    | >= 12         | 其他版本未测试                 |
| npm        | 6.x / 8.x     | 7.x 已确认存在问题             |
| Nacos      | >= 2.x        | 1.3.0 内置 Nacos          |
| Sentinel   | 框架内置          | 直接使用即可                  |
| Seata      | 框架内置          | 直接使用即可                  |

#### 4.2 自建 Nacos

**Nacos 数据库必须指向 `ry-config`（此库为定制配置库，不可替换）。**

将项目 `config/nacos` 下所有配置复制到 Nacos 中。  
建议手动复制内容，避免编码不一致导致解析问题。

### 5. 修改 Nacos 自定义配置

微服务配置较复杂，请确认理解后再修改。

- `application-common.yml`
- `datasource.yml`
- `seata-server.properties`

### 6. 使用内置 `ruoyi-seata-server`

执行 `ry-seata.sql` 初始化 Seata 服务端数据库。  
修改 Nacos 中 `seata-server.properties` 的数据库地址。  
启动 `ruoyi-seata-server` 服务。

### 7. 服务启动顺序

必启基础服务：`MySQL`、`Redis`、`Nacos`  
可选基础服务：`MinIO`（文件上传）、`Seata`（分布式事务，默认开启）、`Sentinel`（熔断限流）、`Monitor`（监控）、`XXL-JOB`（定时任务）

必启应用服务：`Gateway`、`Auth`、`System`  
可选应用服务：`Resource`（文件上传、邮件、短信）、`Gen`（代码生成）、`Job`（定时任务）、`Demo`（演示）
